# CrossTerm — AI Assistant Instructions

This file provides context and rules for AI assistants (Claude, Copilot, etc.) working on the CrossTerm codebase.

## Project Summary

CrossTerm is a **Tauri 2.x** cross-platform terminal emulator & remote access suite. Rust backend + React 18 / TypeScript frontend with WebView rendering.

## Critical Rules

### Architecture

1. Frontend communicates with Rust backend ONLY via `invoke()` and `listen()`. Never access filesystem/process/network directly from frontend.
2. Tauri command naming: `{module}_{verb}` (e.g., `vault_unlock`, `ssh_connect`).
3. Tauri event naming: `{module}:{event-name}` with colon separator (e.g., `terminal:output`).
4. One state struct per Rust module, registered via `tauri::Builder::manage()`.
5. Four Zustand stores: `appStore`, `sessionStore`, `terminalStore`, `vaultStore`.

### Frontend (TypeScript/React)

1. **Function components only.** No class components.
2. **TypeScript strict mode.** No `any`. Use `unknown` + type guards.
3. **ALL types/enums/interfaces** in `src/types/index.ts`. Exception: component-local props.
4. **Zustand selector pattern**: `useAppStore((s) => s.field)`, never `useAppStore()`.
5. **`@/` path alias** for all internal imports. `import type` for type-only imports.
6. **TailwindCSS token classes only**: `bg-surface-primary`, `text-text-secondary`. Never hardcode colors.
7. **`clsx()`** for conditional class names.
8. **All user-facing strings** via `t()` from `useTranslation()`. No hardcoded strings in JSX.
9. **Lucide React** icons at standard sizes (20px rail, 16px actions, 14px inline, 11px status bar).
10. **String enums** with snake_case values matching Rust serde conventions.

### Backend (Rust)

1. **Module layout**: Error → Types → Helpers → State struct → Tauri commands.
2. **Section markers**: `// ── Section Name ──` (visual separation pattern).
3. **Error handling**: `?` operator always. Never `.unwrap()` in command paths.
4. **Serde**: `rename_all = "snake_case"` on enums, `tag = "type"` for tagged unions, `skip_serializing` on secrets.
5. **Security**: AES-256-GCM encryption, `Zeroizing<Vec<u8>>` for keys, UUID v4 for IDs, never log secrets.
6. **Concurrency**: `Mutex` write-heavy, `RwLock` read-heavy, `TokioMutex` for async state.
7. **Error types**: One `thiserror` enum per module. Implement `Serialize` via `Display`.
8. **Every command** returns `Result<T, ModuleError>`.

### Design System

1. **Three-layer token indirection**: Theme JSON → CSS custom properties → Tailwind utilities.
2. **Six-region layout**: TitleBar (A), TabBar (B), Sidebar (C), SessionCanvas (D), BottomPanel (E), StatusBar (F).
3. **4px spacing grid**: All spacing in multiples of 4px.
4. **Empty states**: Always icon + message + optional action. Never blank panels.
5. **Transitions**: Use `--duration-*` CSS variables. Respect `prefers-reduced-motion`.

### Testing

1. Rust: `#[cfg(test)]` inline modules. ≥80% line coverage.
2. Frontend: Vitest for stores/utilities. ≥70% coverage.
3. E2E: Playwright. 20 critical user journeys.
4. Security: `cargo audit`, `npm audit`, `cargo clippy -D warnings` must be clean.

## Common Tasks

### Adding a new Tauri command
1. Add function in the relevant `src-tauri/src/{module}/mod.rs`
2. Register in `lib.rs` `invoke_handler` array
3. Call from frontend via `invoke("module_action", { params })`

### Adding a new event
1. Define payload struct in the Rust module (derive `Clone, Serialize`)
2. Emit via `app_handle.emit("module:event-name", payload)`
3. Listen in frontend: `listen<PayloadType>("module:event-name", callback)`

### Adding a new UI string
1. Add key to `src/i18n/en.json`
2. Use via `t("domain.key")` in component

### Adding a new theme token
1. Add to `ThemeTokens` interface in `src/types/index.ts`
2. Add CSS variable in `src/index.css` (both `:root` and `.light`)
3. Add Tailwind mapping in `tailwind.config.ts`
4. Add values in `src/themes/{dark,light}.json`

### Adding a new component
1. Create `src/components/{Feature}/{ComponentName}.tsx`
2. Use function component, PascalCase filename
3. Import stores via selectors, types from `@/types`
4. Use `t()` for all strings, token classes for styling
